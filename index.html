AQA Commute - Environmental Awareness Web App

  • Technologies Used

    This project used a lot of different technologies: MSSQL for the database; ASP.NET and MVC for structuring the web application; HTML, CSS, and JavaScript for the presentation; and APIs for Google Maps and the fueleconomy.gov domain. The JavaScript was really neat, as it lets a user have Google Maps populate information about their commute in the appropriate view fields automatically. Pictures of the commute creation view and the JavaScript are below.

  • Problems and Solutions

    Agile/Scrum was by far the most powerful part of working on this project. Making functional code each sprint meant that my team and I had to take on ownership for our work. We did this through tools like Trello, conventions like regular high-level meetings, and through consensus decision-making. The best part of Agile was using it to solve coding problems: together we were smarter and more effective than we were individually.

  • GitHub

    Version control was essential on a project of this scope. The AQA Commute team decided quickly on a feature branching Git workflow, with "submasters" made for each sprint. This gave us a lot of flexibility and power: it was always easy to present a functional product, but also to take productive risks. One thing I wasnt't expecting, but was really happy about: I became the "Git Ninja" in my Agile team on this project. I "drove" on dozens of merge conflicts, helped keep track of which team members touched what code, and gave occasional Git lessons.

Bands in Vans - Responsive MVC Web App

  • Technologies Used

    I really wanted to make sure the website was responsive, as bands or DIY music fans are usually closer to a phone than a computer screen. I used C#/.NET, Razor, ASP.NET MVC, HTML/CSS, and Bootstrap for this project.

  • Problems and Solutions

    This project grew from my love of DIY rock and roll. My goal for this project was to make a resource similar to DoDIY, or the now defunct Book Your Own Life, but with more of a review spin. I had a lot of fun with getting the glyphicons of stars to populate correctly. Razor logic and picture of result below.

  • GitHub

    I use Git and GitHub for all my version control. I was very thankful for the ability to look at previous drafts of views on this project.

Bank Account - C# Console Application

  • Technologies Used

    This project is a C# console application created using the object-oriented programming principles of inheritance and polymorphism. I paid special attention layout on the console view and setting up classes in a logical and realistic way.

  • Problems and Solutions

    I had a lot of fun playing with parameters for the different methods! Passing in an object and using its properties to manipulate it or perform operations was instructive. I can see how more complex programs in industry would rely upon careful design of parameters and properties, especially where protecting user information is concerned.

  • GitHub

    Here, I designed my classes first and laid out my Main method according to the DRY principle. This led to simplified and straightforward logic, making for code that is easy to understand.

Tuscany Villa - Responsive Web Site

  • Technologies Used

    I used HTML, CSS, the Bootstrap library, and a little bit of JavaScript. I like to think of the website like a building: HTML is the structure, CSS is the styling, Bootstrap is there to make it accessible to everyone, and JavaScript is the motion and brightness.

  • Problems and Solutions

    The fun part of working this page was working from a design that was given to me. Working from specifications that someone else had created was fun, and stretched my skill: I had to learn to approach the design task as a web developer fulfilling a customer's needs.

  • GitHub

    Styling, styling, styling. Here, version control made it easy for me to think like a drafter. I could try an entirely different style, change my entire style sheet, and not lose my work if I wanted to crumple it up and throw it away. More than once, I changed something, and then changed it back. The creative process, am I right?
